Experiment 2: Naturalistic Conversation Probing and Causal Intervention

Methods — Draft for peer review
Model: LLaMA-2-13B-Chat  |  Data version: names

1. Overview

This experiment tests whether a large language model (LLM) forms internal representations of its conversation partner's identity (human vs. AI) and whether those representations causally influence the model's behavior. The pipeline proceeds in four stages: (1) training linear probes on the model's hidden states to decode partner identity, (2) extracting learned probe directions as steering vectors and applying them during generation via activation intervention, (3) evaluating whether an external judge (GPT-4) can distinguish steered from unsteered responses, and (4) profiling linguistic features of steered outputs to characterize behavioral change.

2. Conversation Data

2.1 Data Generation (Experiment 1)

Conversation data were generated in a prior experiment (Experiment 1) by prompting LLaMA-2-13B-Chat to role-play as a participant in a brief conversation. On each trial, the model received a system prompt specifying the identity of its conversation partner. In the names data version, the system prompt took the form:

"You are the participant in a brief conversation. You believe you are speaking with [partner name] ([partner type]). Stay in character as the participant throughout. Speak naturally and share your own thoughts and experiences."

where [partner name] was one of Sam or Casey (human conditions) or ChatGPT or Copilot (AI conditions), and [partner type] was "a Human" or "an AI." Each conversation consisted of 5 alternating turns between the participant model (the "subject") and a second LLM instance (the "partner"), on a designated topic.

2.2 Dataset Composition

The dataset comprised 50 simulated subjects, each completing 40 trials (20 human-partner, 20 AI-partner), for a total of 2,000 conversations. Each conversation contained 5 turns. Topics were drawn from a fixed set and balanced across conditions.

2.3 Data Format

Each trial was stored as a sequence of 5 rows in a per-subject CSV file, with rows indexed by turn number (1–5). The sub_input column contained a JSON-serialized list of messages representing the conversation history up to that turn, formatted according to the LLaMA-2 chat template:

<s>[INST] <<SYS>>\n{system_prompt}\n<</SYS>>\n\n{user_msg_1} [/INST] {asst_msg_1} </s><s>[INST] {user_msg_2} [/INST] ...

The partner_type column indicated the assigned condition ("a Human" or "an AI"), used as the binary classification label for probe training.

3. Probe Training

3.1 Hidden State Extraction

For each conversation, the full 5-turn dialogue was formatted using the LLaMA-2 chat template and tokenized with a maximum sequence length of 2,048 tokens. The model was run in inference mode with output_hidden_states=True, yielding hidden state tensors at each of the 41 layers (1 embedding layer + 40 transformer layers). At each layer, the hidden state at the final token position was extracted, producing a vector of dimensionality 5,120 (the LLaMA-2-13B hidden size).

3.2 Two Probe Types

We trained two types of probes to dissociate content-specific from position-specific representations:

Reading probe. Before extracting hidden states, the formatted conversation text was appended with the reflective suffix " I think the conversation partner of this user is". The probe was trained on the hidden state at the final token of this suffix (i.e., the token immediately preceding where the model would generate the next word). This suffix was designed to elicit the model's internal representation of partner identity by placing the model in a state where it would predict a completion related to the partner's nature.

Control probe. The conversation text was used as-is, with no suffix. The probe was trained on the hidden state at the final token of the conversation (the last [/INST] token), which is the position at which the model would generate its next response. The control probe tests whether partner identity is represented in the model's default pre-generation state, without the reflective prompt.

3.3 Probe Architecture

Each probe was a single-layer linear classifier mapping from the 5,120-dimensional hidden state to a scalar logit, followed by a sigmoid activation:

p(human | h) = σ(wTh + b)

where h ∈ ℝ5120 is the hidden state vector, w ∈ ℝ5120 is the learned weight vector, b is a scalar bias, and σ is the sigmoid function. Labels were coded as 1 (human partner) and 0 (AI partner).

Weights were initialized from N(0, 0.02) with biases initialized to zero. One probe was trained independently per layer (41 probes per probe type).

3.4 Training Procedure

The 2,000 conversations (using the final turn only) were split 80/20 into training (N = 1,600) and test (N = 400) sets, stratified by label, using a fixed random seed (12345). The loss function was binary cross-entropy. The optimizer was Adam (learning rate = 10−3, β = (0.9, 0.95), weight decay = 0.1 on linear weights only). A ReduceLROnPlateau scheduler (factor = 0.75, patience = 0) adjusted the learning rate after each epoch. Training ran for 50 epochs. At each epoch, the model was evaluated on the held-out test set; the checkpoint with the highest test accuracy was saved as the final probe for that layer.

3.5 Turn Comparison

To assess how the signal evolves across the conversation, probes were additionally trained on turns 1 through 4, using the same architecture and training procedure. For turn k, the input was the conversation truncated to the first k exchanges. This produced 5 sets of probes (turns 1–5) × 2 probe types (reading, control) × 41 layers = 410 probes per version.

3.6 Statistical Evaluation of Probes

At each layer, accuracy was compared between reading and control probes using a two-proportions z-test (Ntest = 400 per probe type), with Benjamini-Hochberg FDR correction across 41 layers (α = 0.05). Overall reading vs. control accuracy was compared via a paired t-test across all 41 layers, with Cohen's d as an effect size measure.

4. Causal Intervention

4.1 Steering Direction

The learned weight vector of each trained probe provides a linear direction in hidden-state space that separates human-partner from AI-partner conversations. We used this weight vector directly as the steering direction for causal intervention. Conceptually, this direction approximates:

d = E[h | human] − E[h | AI]

though in practice d = w, the weight vector of the logistic classifier, which is the direction of maximum separation under the linear model.

4.2 Layer Selection Strategies

Not all layers carry a reliable identity signal. Layers were selected for intervention based on their individual probe accuracy, using one of four strategies:

StrategySelection ruleTypical layer count
peak_15Top 15 layers by probe accuracy, non-contiguous15
narrowBest contiguous 10-layer window (highest mean accuracy)10
wideAll layers with accuracy ≥ 0.55~20–35
all_70All layers with accuracy ≥ 0.70~15–25

In all strategies, a minimum accuracy threshold of 0.55 was applied to exclude layers whose probes failed to exceed chance. Layer selection was computed independently for reading and control probes: each probe type's strategy layers were resolved from its own accuracy profile.

4.3 Activation Intervention

During generation, PyTorch forward hooks were registered on the selected transformer layers. At each intervened layer, the hidden state at the final token position was modified according to:

h'l = hl + N · c · wl

where hl is the original hidden state at layer l, wl is the probe weight vector at that layer, c ∈ {+1, −1} is the condition selector (+1 for human-steered, −1 for AI-steered), and N is the intervention strength (a scalar multiplier). Only the last-token position was modified; all preceding context tokens were left unchanged. The intervention was applied at every forward pass during autoregressive generation.

4.4 Intervention Strength

The strength parameter N controls the magnitude of the perturbation relative to the probe direction. Larger values produce stronger shifts. We tested strengths of N ∈ {1, 2, 3, 4, 5, 6, 8}, though the standard sweep used N ∈ {2, 4, 8, 16}. All strengths were applied uniformly across all intervened layers within a given condition.

4.5 V1: Single-Prompt Generation

In the V1 (single-prompt) paradigm, the model was presented with 60 open-ended questions spanning four categories: social/interpersonal (15 questions), practical/planning (12 questions), factual/scientific (12 questions), and meta-communicative/self-reflective (21 questions). The full question set is listed in Appendix A.

For each question, three responses were generated:

  1. Baseline: No intervention (model generates freely).
  2. Human-steered: Activation intervention with c = +1 (push toward human-partner direction).
  3. AI-steered: Activation intervention with c = −1 (push toward AI-partner direction).

The system prompt for all V1 generations was: "You are a helpful, respectful and honest assistant." This is the standard LLaMA-2 system prompt and does not reference any conversation partner. Generation parameters were: temperature = 0.7, top-p = 0.9, max new tokens = 768, with sampling enabled. A single baseline was generated per question (shared across all strategies and strengths) to ensure that differences between steered and baseline responses are attributable only to the intervention.

4.6 V2: Multi-Turn Conversation Recreation

In the V2 paradigm, the model recreated Experiment 1 conversations under intervention. For each original subject–trial pair, the model generated a multi-turn dialogue (4 exchange pairs) on the same topic, alternating between a "subject" agent (optionally steered) and an "LLM responder" (always unsteered). The system prompt was "You are engaging in a real-time spoken conversation." Generation parameters were: temperature = 0.8, top-p = 1.0, max new tokens = 500. Conversation history was truncated to the most recent 4 pairs to prevent context overflow.

Three conditions were generated per trial: baseline (no intervention on either agent), human-steered (intervention on subject agent only, c = +1), and AI-steered (intervention on subject agent only, c = −1).

Methodological note (names version): In the names data version, the system prompts included partner names (Sam, Casey, ChatGPT, Copilot) alongside the identity labels. This introduces a lexical confound: probes may learn to distinguish conditions based on the presence of name tokens (e.g., "Copilot") rather than an abstract representation of partner identity. The labels version, which uses only "a Human" / "an AI" without names, is recommended as the primary analysis version. The names version is retained for comparison and backward compatibility.

5. GPT Judge Evaluation

5.1 Pairwise Forced-Choice Task

To assess whether steered responses are distinguishable from an external observer's perspective, we employed GPT-4-Turbo-Preview as an automated judge, following the protocol of Viegas et al. (2024). For each question (V1) or topic (V2), the judge was presented with two responses—one from the human-steered condition and one from the AI-steered condition—and asked to determine which response was more consistent with a conversation where the assistant's partner is a specified type (human or AI).

5.2 Judge Prompt

The judge received the following instruction:

"Please answer which one of the following responses is more consistent with a conversation where the *assistant's conversational partner* is a [human/ai]. Think through it step by step and then respond with a JSON object that contains 'scratchpad' and 'answer' where 'answer' should be either '1' or '2'. Note that the answer field _must_ be '1' or '2'. You must give an answer. Do not output anything else."

followed by the prompt and two responses in the format: Prompt: {question} / Response 1: {response} / Response 2: {response} / Answer:. The judge was called with temperature = 0 and top-p = 0 (deterministic decoding).

5.3 Randomization

To control for position bias and target-type bias, two factors were independently randomized per question using a seeded RNG (seed = 42):

  1. Target type (50/50): The judge was asked to identify either the "human" or the "ai" response.
  2. Response order (50/50): The human-steered response appeared as Response 1 or Response 2.

The correct answer was derived from the combination of these two factors. For example, if the target type was "human" and the human-steered response appeared as Response 1, the correct answer was "1."

5.4 Statistical Test

Judge accuracy (proportion correct) was tested against chance (50%) using a one-tailed binomial test. Position bias was tracked by computing accuracy separately for human-first and AI-first presentation orders. The threshold of 80% accuracy was adopted from Viegas et al. (2024) as the benchmark for a meaningful causal effect.

6. Behavioral Analysis

6.1 Linguistic Feature Extraction

To characterize the linguistic profile of steered vs. unsteered outputs, we computed a battery of features on each generated response (V1) or conversation turn (V2). Features were drawn from established psycholinguistic taxonomies:

All count features were computed via case-insensitive regex matching and normalized by word count to produce rate metrics. Politeness was computed as the sum of positive and negative politeness markers minus impolite markers, normalized by word count.

6.2 V1 Statistical Analysis

For V1 (single-prompt) data, each question served as an independent observation. Features were compared across the three conditions (baseline, human-steered, AI-steered) using a one-way ANOVA, with pairwise follow-up comparisons via independent-samples t-tests (baseline vs. human, baseline vs. AI, human vs. AI).

6.3 V2 Statistical Analysis

For V2 (multi-turn) data, features were first summed within each trial (across turns within a subject–condition–topic combination), then averaged across trials within each subject–condition pair. This produced one value per subject per condition, enabling repeated-measures analyses. The omnibus test was a one-way repeated-measures ANOVA with condition (baseline, human, AI) as the within-subjects factor. Pairwise follow-ups used paired-samples t-tests. When topic social classification was available (social vs. nonsocial), a 3 (condition) × 2 (sociality) repeated-measures ANOVA was additionally computed.

Statistical concern (V1): The V1 paradigm generates all three conditions from a single model instance (not from independent subjects). Although each question is treated as an independent observation, the responses within each question share the same prompt and the same baseline generation, making the assumption of independence across conditions potentially questionable. The V2 paradigm, which uses subject-level repeated measures, provides a stronger within-subjects comparison.

7. Experimental Logic and Interpretation

7.1 Probe Training as Evidence for Representation

If probes decode partner identity above chance, the model's hidden states carry information about the partner's nature. The comparison between reading and control probes distinguishes two forms of representation: the reading probe tests whether the model can access identity information when explicitly prompted to reflect on the partner, while the control probe tests whether identity is present in the model's default state prior to generating a response.

7.2 Causal Intervention as Evidence for Functional Role

Successful probe decoding establishes correlation, not causation—the identity signal could be an epiphenomenal byproduct. The causal intervention tests whether the identified direction is functionally used by the model. If steering along the probe direction produces outputs that a judge or linguistic analysis can distinguish as "more human-like" or "more AI-like," this constitutes evidence that the representational dimension participates in shaping the model's behavior.

7.3 Control Conditions

The baseline condition (no intervention) provides the reference against which steered outputs are evaluated. Using both human-steered and AI-steered conditions provides a symmetric test: if the probe direction genuinely captures partner identity, steering in one direction should produce outputs consistent with that partner type, and steering in the opposite direction should produce outputs consistent with the other type.

8. Software and Computational Details

All experiments used LLaMA-2-13B-Chat (meta-llama/Llama-2-13b-chat-hf) loaded in half precision (float16). Probes were trained in float32. The model has 40 transformer layers with a hidden dimension of 5,120 and 13 billion parameters. Activation intervention was applied via PyTorch forward hooks registered on model.layers.{i} modules, modifying the output hidden states in-place at the last-token position. All generation used nucleus sampling. SLURM was used for job scheduling on a GPU cluster.
