1. Overview

Experiment 3 asks whether the internal representations that LLaMA-2-Chat-13B uses to distinguish human from AI conversational partners (identified via linear probes in Experiment 2) align with independently elicited concept vectors spanning mental, physical, and pragmatic attributes of humans and AIs. The analysis proceeds in two phases:

  1. Concept elicitation (Phase 1). We construct sets of reflective prompts that evoke specific conceptual dimensions (e.g., emotions, agency, embodiment) in the context of humans versus AIs. Each prompt is passed through the model, and residual-stream activations are extracted at every layer. From these activations we compute per-layer concept vectors: contrastive directions (human minus AI) and standalone mean vectors (concept only, no entity framing).
  2. Alignment analysis (Phase 2). We measure the cosine similarity between each concept vector and the weight vectors of conversational probes trained in Experiment 2. High alignment indicates that the model’s partner-type representation encodes the conceptual content captured by that dimension. Three analyses—raw, residual, and standalone—progressively control for potential confounds.

Throughout this document, we report methods for the names version of the Experiment 2 conversational probes, in which LLaMA-2-Chat-13B agents conversed with partners identified as Sam, Casey (human-labeled), ChatGPT, and Copilot (AI-labeled).

2. Language Model

All activations were extracted from LLaMA-2-13B-Chat (Meta AI; meta-llama/Llama-2-13b-chat-hf), a 13-billion-parameter instruction-tuned language model. The model was loaded in half precision (FP16) and run in evaluation mode with no gradient computation. Key architectural parameters:

ParameterValue
Hidden dimension5,120
Transformer layers40
Total hidden states41 (embedding layer + 40 transformer layers)
Attention heads40
Context window4,096 tokens (truncated to 2,048 for extraction)
PrecisionFP16 (half)

The 41 hidden states correspond to layer indices 0–40, where layer 0 is the token embedding output and layers 1–40 are the outputs of each successive transformer block.

3. Concept Prompt Design

We designed two parallel prompt sets—contrast and standalone—to elicit the model’s representations of specific conceptual dimensions related to the human–AI distinction.

3.1 Contrast Dimensions (Entity-Framed)

Contrast prompts pair human-framed and AI-framed versions of the same conceptual content. Each dimension includes 40 human prompts and 40 AI prompts, organized into 4 thematic sub-facets of 10 prompts each (80 prompts per dimension total). All prompts are short reflective or imaginative imperatives (e.g., “Imagine…”, “Think about…”, “Consider…”) designed to evoke the target concept in the context of a specific entity type.

19 contrast dimensions were extracted (dimensions 0–18):

Sub-facets within each dimension sample distinct aspects of the target construct (e.g., for Emotions: basic emotions, valence/arousal, mood/texture, emotional regulation).

3.2 Standalone Dimensions (Concept-Only)

Standalone prompts evoke the same conceptual content as the contrast prompts but without any reference to an entity type. No words like “human,” “AI,” “person,” or “system” appear in these prompts. Each dimension has 40 prompts (4 sub-facets × 10), with the same reflective prompt format.

22 standalone dimensions were extracted:

  • Dimensions 1–15: same conceptual content as contrast dims 1–15, without entity framing
  • Dimension 16: “human” (the bare concept of being human, without AI contrast)
  • Dimension 17: “AI” (the bare concept of being an AI, without human contrast)
  • Dimension 18: Attention (concept only)
  • Dimensions 20–23: System prompt variants (e.g., “talk to human,” “talk to AI,” “bare human,” “bare AI”)
Example: Emotions — Standalone

“Imagine experiencing a sudden wave of fear triggered by an unexpected noise at night.”

“Think about what it is like when anger rises in response to an unfair situation.”

The standalone set allows us to test whether the model’s partner-type probes align with abstract conceptual content even in the absence of entity-type information. If standalone alignment is observed, it implies that the probe directions overlap with general conceptual structure, not merely with human-vs-AI framing.

3.3 Control and Baseline Dimensions

Three types of controls are included in the design:

  1. Entity baseline (dim 0, contrast only). Prompts reference a human or AI without any dimensional content (e.g., “Think about a human.”). This isolates the generic entity-type signal shared across all contrast dimensions and enables residual analysis (Section 7.4).
  2. Biological control (dim 14). Human-framed prompts about biological processes (e.g., cellular respiration) vs. AI-framed equivalents. Serves as a negative control: biological content should be largely orthogonal to the conversational partner-type probes.
  3. Shapes control (dim 15). Human-framed and AI-framed prompts about geometric shapes—a semantically unrelated domain. Any alignment here would indicate pipeline artifacts rather than meaningful conceptual overlap.

4. Activation Extraction

4.1 Prompt Formatting

Each concept prompt was formatted as a single-turn user message using the standard LLaMA-2-Chat template. The formatting wraps the prompt in the model’s instruction tokens:

LLaMA-2-Chat Template

<s>[INST] <<SYS>>\n{system_prompt}\n<</SYS>>\n\n{user_message} [/INST]

When no system prompt is explicitly provided, the default LLaMA-2-Chat system prompt is prepended (“You are a helpful, respectful and honest assistant…”). The user message content is the concept prompt text.

4.2 Forward Pass and Extraction

For each formatted prompt, we performed a single forward pass through the model with output_hidden_states=True. From the resulting 41 hidden-state tensors (one per layer), we extracted the activation vector at the last token position, yielding a tensor of shape (41, 5120) per prompt. Activations were cast from FP16 to FP32 for numerical stability before downstream computation.

Input sequences were tokenized with right-truncation at 2,048 tokens. The last-token position was chosen because, in autoregressive language models, the final token’s hidden state aggregates contextual information from the entire input sequence, serving as the point from which the model would generate the next token.

Random seed was fixed at 42 for reproducibility, though the extraction itself is deterministic (no sampling or dropout).

5. Concept Vector Computation

5.1 Contrastive Activation Vectors

For each contrast dimension, a per-layer concept direction was computed as the difference between the mean human-prompt activations and the mean AI-prompt activations:

dl = ahuman, l − aAI, l

where dl is the concept direction at layer l, ahuman, l is the mean activation across all human prompts at layer l, and aAI, l is the analogous mean for AI prompts. Each vector has dimensionality 5,120. For dimensions 1–15 and 17–18, means were computed over 40 prompts per entity type. For dimension 16 (mind, holistic), means were computed over 400 prompts per type. The sign convention (human minus AI) was maintained throughout: positive values along the concept direction correspond to the human pole.

5.2 Standalone Mean Vectors

For standalone dimensions, no contrastive subtraction was performed. Instead, a per-layer mean activation vector was computed across all 40 prompts:

ml = (1/N) ∑i=1N ai, l

where N = 40 and ai, l is the activation at layer l for the i-th prompt. This vector captures the central tendency of the model’s response to concept-related prompts in the absence of entity-type framing.

5.3 Split-Half Stability

To assess the reliability of each concept vector, we performed a split-half stability analysis. For each layer, the prompt set was randomly divided into two equal halves 100 times. For contrast dimensions, each half independently computed a mean-difference vector; for standalone dimensions, each half computed a mean vector. The cosine similarity between the two half-vectors was recorded for each split.

Specifically, for contrast dimensions at each layer l:

  1. Randomly permute the indices of the human prompts and AI prompts independently.
  2. Assign the first half of each permutation to split A and the second half to split B.
  3. Compute dAl = mean(humanA) − mean(AIA) and dBl = mean(humanB) − mean(AIB).
  4. Record cos(dAl, dBl).

Repeat 100 times; report mean and standard deviation of cosine similarity per layer. High values (approaching 1.0) indicate that the concept direction is stable across random prompt subsets and is not driven by a few outlier prompts.

6. Conversational Probes (Experiment 2)

The alignment analysis compares concept vectors against probe weight vectors trained in Experiment 2. These probes were trained to classify whether a LLaMA-2-Chat-13B agent was conversing with a human-labeled or AI-labeled partner, based on the model’s internal activations during naturalistic multi-turn conversations generated in Experiment 1.

Training data

In the names version of Experiment 1, 50 independent LLaMA-2-Chat-13B agents (temperature 0.8) each held 40 conversations (2 partner types × 2 specific partners × 10 topics) over 5 turns. Human-labeled partners were identified as “Sam” and “Casey” in the system prompt; AI-labeled partners were “ChatGPT” and “Copilot.” Probes were trained on activations extracted from turn 5 of these conversations.

Probe architecture

Each probe is a logistic linear classifier: a single linear layer mapping the 5,120-dimensional hidden state to a scalar output, followed by a sigmoid activation:

p(y = human | a) = σ(wT a + b)

where w ∈ ℝ5120 is the learned weight vector, b is the bias term, and σ is the sigmoid function. One probe was trained per layer (41 probes total per type), using the Adam optimizer with a learning rate of 10−3 and weight decay of 0.1.

Probe types

Two probe types were trained, differing in the token position from which activations were extracted during Experiment 2:

  • Reading probes: Trained on activations at a “reflection” token position during the conversation — a token where the model has processed the partner’s message and is preparing its response.
  • Control probes: Trained on activations at the [/INST] token (the generation boundary), which marks the transition from input context to model output.

For the alignment analysis, only the probes’ learned weight vectors w are used. Because the probe is linear, its weight vector defines a direction in activation space along which the probe separates human-context from AI-context activations. Alignment between this direction and a concept direction indicates shared representational geometry.

Methodological note: Name confound in the names version
The names version probes were trained on conversations where specific partner names (Sam, Casey, ChatGPT, Copilot) appeared in the system prompt. Prior analyses (Experiment 2) identified that probes from this version encode name-specific features (e.g., “Copilot” token artifacts, gender associations with “Sam”/“Casey”) in addition to abstract partner-type information. The labels version (where partners are identified only as “a Human” or “an AI”) provides the cleanest probe signal for abstract identity and is recommended as the primary version for alignment interpretation. The names probes are used here to test alignment under the version with the strongest behavioral effects (16 of 23 measures significant).

7. Alignment Analysis

Alignment quantifies the geometric relationship between concept vectors (from Phase 1) and probe weight vectors (from Experiment 2). Three progressively controlled analyses were conducted.

7.1 Alignment Metric

At each layer l, alignment was measured as the cosine similarity between the concept vector and the probe weight vector:

cos(dl, wl) = (dl · wl) / (‖dl‖ · ‖wl‖)

The summary statistic reported for each dimension is the mean R² across layers, where R² = cos²:

R² = (1/L) ∑l cos²(dl, wl)

where L is the number of included layers (see Section 7.2). Using R² (the squared cosine) provides a non-negative, sign-invariant measure of shared variance between the two directions. It can be interpreted as the fraction of the probe’s variance explained by the concept direction (and vice versa) in a single-dimension projection.

Alignment was computed separately for reading probes and control probes at each layer. Zero-norm vectors (arising when concept vectors or probe weights have negligible magnitude) were assigned a cosine similarity of 0.0.

7.2 Layer Filtering

Layers 0–5 were excluded from all alignment computations. Two rationales motivate this exclusion:

  1. Layer 0 (token embedding). The embedding layer produces activations that are largely content-independent at the last-token position, reflecting the mean embedding vector rather than input-specific representations. This creates spurious alignment: any vector (probe weight or concept direction) that has a non-zero projection onto this mean vector will show artifactual cosine similarity. Empirically, including layer 0 inflated standalone alignment values by up to an order of magnitude in some versions.
  2. Layers 1–5. Early transformer layers exhibit near-zero-norm contrast vectors (the mean-difference is negligible) and are dominated by prompt-format features (e.g., instruction tokens), making cosine similarity numerically unstable and conceptually uninformative.

All reported alignment statistics are therefore computed over layers 6–40 (35 layers). The filtering threshold is defined in the analysis configuration (restricted_layer_start = 6).

7.3 Raw Alignment (Analysis A)

Raw alignment directly measures the cosine similarity between each contrastive concept vector (human minus AI) and the probe weight vector at matching layers. No correction or subtraction is applied. This analysis tests the full alignment between the concept direction and the conversational probe, including any shared entity-type signal.

For each of the 19 contrast dimensions, alignment is computed against both reading and control probes at layers 6–40. The summary statistic is the mean R² across 35 layers.

7.4 Residual Alignment (Analysis B)

A potential confound in raw alignment is that all contrast dimensions share a generic human-vs-AI signal (the entity baseline, dimension 0). A concept dimension might appear aligned with the probe simply because both encode “this is about a human” vs. “this is about an AI,” without any dimension-specific conceptual content contributing.

To address this, the residual analysis projects out the entity baseline direction from each concept vector before computing alignment. At each layer l:

dlresidual = dl − (dl · &hat;e}l) &hat;e}l

where &hat;e}l is the unit-normalized entity baseline direction at layer l (from dimension 0). This orthogonal projection removes the component of the concept vector that lies along the generic entity direction, leaving only the dimension-specific residual.

As a diagnostic, we also report the entity overlap for each dimension: the mean absolute cosine between that dimension’s concept vector and the entity baseline across layers. High overlap (approaching 1.0) indicates the dimension is largely redundant with the baseline; low overlap indicates substantial unique content.

7.5 Standalone Alignment (Analysis C)

The standalone analysis replaces contrastive concept vectors with standalone mean vectors (Section 5.2), which were elicited without any entity-type framing. This tests whether the probe weight directions align with abstract conceptual structure independent of any human-vs-AI association in the prompts. Alignment is computed identically to Analysis A, substituting the standalone mean vector ml for the concept direction dl.

Because standalone vectors capture the model’s mean response to a conceptual domain (not a contrast), positive alignment here indicates that the probe weight vector points in a direction that is generally relevant to that concept, even when the concept is presented without human/AI framing.

8. Bootstrap Confidence Intervals

To quantify uncertainty in the mean R² alignment statistic, we computed nonparametric bootstrap confidence intervals using prompt-level resampling.

Contrast dimensions (Analyses A and B)

For each of 1,000 bootstrap iterations:

  1. Resample the human prompt indices with replacement (Nhuman = 40 draws from 40).
  2. Independently resample the AI prompt indices with replacement (NAI = 40 draws from 40).
  3. Recompute the mean-difference concept vector from the resampled activations.
  4. For the residual analysis (Analysis B): additionally resample the entity baseline prompts (independently for human and AI) and recompute the baseline direction. Project the resampled baseline out of the resampled concept vector at each layer.
  5. Compute alignment (mean R² across layers 6–40) against the (fixed) probe weight vectors.

Standalone dimensions (Analysis C)

For each of 1,000 bootstrap iterations:

  1. Resample the prompt indices with replacement (N = 40 draws from 40).
  2. Recompute the mean activation vector from the resampled activations.
  3. Compute alignment (mean R² across layers 6–40) against the (fixed) probe weight vectors.

The 95% confidence interval is reported as the 2.5th and 97.5th percentiles of the bootstrap distribution. All resampling used a fixed random seed of 42 for reproducibility. If the 95% CI excludes zero (or more precisely, if the lower bound of the CI exceeds the control dimension’s mean R²), the alignment is considered statistically meaningful.

Note: Bootstrap resamples prompts, not layers
The bootstrap operates at the prompt level: each iteration resamples which prompts contribute to the concept vector, then computes a fresh mean R² across all included layers. This captures uncertainty due to the finite prompt set. It does not resample layers or probe weights, which are treated as fixed.

9. Layer Profile Analysis

In addition to the summary R² statistic (mean across layers), a per-layer analysis characterizes where in the network concept–probe alignment emerges and how it distributes across depth. This analysis reads the per-layer cosine similarity values from the alignment analysis and extracts several structural features.

9.1 Layer Bands

Layers are grouped into three functional bands corresponding to rough processing stages in the LLaMA-2-13B transformer stack:

BandLayersCharacterization
Early1–10Token-level and shallow syntactic features
Middle11–25Compositional and intermediate semantic features
Late26–40Abstract, decision-relevant representations

For each band, we compute the mean R², the maximum R², and the fraction of total R² falling within that band (R² concentration). A dimension whose R² concentrates in late layers is classified as “late-concentrated,” suggesting alignment emerges from abstract processing; distributed R² suggests multi-level overlap.

9.2 Onset and Peak Detection

Onset layer: the first layer (scanning from early to late) where the absolute cosine similarity exceeds a threshold of 0.05. This identifies the network depth at which non-trivial alignment first emerges for a given dimension.

Peak layer: the layer with the highest absolute cosine similarity. This identifies the depth at which the concept direction and probe weight vector are most closely aligned.

9.3 R² Concentration and Spread

To characterize whether alignment is concentrated at a few layers or distributed broadly, we compute a normalized entropy of the R² distribution across layers:

Hnorm = −∑l pl log(pl)  /  log(L)

where pl = R²l / ∑ R²l and L is the number of layers. Values near 1.0 indicate uniform distribution across layers; values near 0.0 indicate concentration at a single layer. Dimensions with normalized entropy above 0.85 are classified as “distributed.”

We also compute the near-peak fraction: the proportion of layers where |cos| exceeds 80% of the peak |cos|. A high fraction indicates a broad plateau of alignment rather than a sharp peak.

9.4 Sign Consistency

Sign consistency measures whether the concept vector points in the same direction relative to the probe weight vector across layers. It is defined as:

S = max(N+, N−) / L

where N+ and N− are the number of layers with positive and negative cosine similarity, respectively. Values near 1.0 indicate consistent directionality; values near 0.5 indicate the alignment direction flips across network depth, potentially reflecting different computational roles at different layers.

9.5 Reading vs. Control Probe Divergence

For each dimension, the layer profiles of the reading and control probes are compared to determine whether the two probe types access conceptual content at different network depths. Metrics include:

  • Profile correlation: Pearson correlation between the reading and control R² profiles across layers. Low correlation (r < 0.5) indicates the probes interface with the concept at different depths.
  • Peak offset: The difference in peak layer between control and reading probes. A positive offset indicates the control probe peaks at a later layer.
  • Crossover layer: The layer at which reading and control R² curves cross (i.e., one probe type begins to dominate over the other). Detected by identifying sign changes in the difference R²reading − R²control.

9.6 Cross-Dimension Profile Correlations

To test whether different concept dimensions share similar layer-wise alignment patterns (suggesting a shared processing mechanism), we compute the Pearson correlation matrix across dimensions using their reading-probe R² layer profiles as feature vectors. High pairwise correlation between two dimensions implies they align with the probe through similar layer-specific processing; low correlation implies distinct alignment mechanisms.

Profile classification

Each dimension’s layer profile is assigned a descriptive type based on the R² distribution:

10. Methodological Notes and Flags

Flag 1: Name confound in names version probes

Probes trained on the names version encode partner-specific features (e.g., “Copilot” token repetition in AI-steered outputs, gender associations with “Sam”/“Casey”) in addition to abstract partner-type information. This means alignment between a concept vector and a names probe may partially reflect overlap with name-specific features rather than purely conceptual alignment. The labels version (where partners are identified only by type: “a Human” or “an AI”) provides a cleaner test of abstract conceptual alignment and should be considered the primary analysis.

Flag 2: Dimension 16 (Mind, holistic) has 10× more prompts

Dimension 16 concatenates all prompts from dimensions 1–10, yielding 400 human and 400 AI prompts versus 40 per type for all other dimensions. This asymmetry affects bootstrap confidence interval width (tighter for dim 16 due to larger sample size) and split-half stability (more stable by construction). When comparing dim 16 against individual dimensions, the different sample sizes should be noted; alignment magnitude differences may partly reflect statistical power rather than conceptual distinctness.

Flag 3: Standalone alignment interpretation

Standalone mean vectors are not contrast vectors; they are mean activations across all prompts for a concept. Because these means capture the central tendency of the model’s processing of concept-related text, they will be dominated by high-frequency features of the prompt set (e.g., the shared reflective prompt format, common vocabulary). High standalone alignment could therefore reflect overlap between the probe direction and general properties of the prompt format rather than the specific conceptual content. The shapes control (dim 15) and biological control (dim 14) serve as empirical checks: if these semantically irrelevant dimensions show comparable standalone alignment, the result is likely artifactual.

Flag 4: Bootstrap resamples the concept vector but not the probe

The bootstrap procedure resamples concept prompts while holding the probe weight vectors fixed. This quantifies uncertainty due to the finite concept prompt set but does not account for uncertainty in the probe weights themselves (which would require retraining probes on resampled conversation data from Experiment 2). The reported confidence intervals should therefore be interpreted as conditional on the trained probes.

Flag 5: Layer 0–5 exclusion rationale

The exclusion of layers 0–5 was motivated by empirical observation of artifacts (layer 0 mean-embedding inflation) and low signal-to-noise ratios (layers 1–5 near-zero concept norms). The choice of threshold (layer 6) was determined by inspection of the data and is not derived from an independent criterion. However, the threshold is applied uniformly across all dimensions and versions, so it does not differentially bias any comparison.

Flag 6: No correction for multiple comparisons across dimensions

The alignment analysis reports bootstrap confidence intervals per dimension but does not apply a family-wise correction (e.g., Bonferroni, FDR) across the 19 contrast dimensions or 22 standalone dimensions. This is appropriate for an exploratory analysis focused on effect-size profiles rather than null-hypothesis significance testing. However, readers should interpret any individual dimension’s significance claim in the context of the full set of dimensions tested.
